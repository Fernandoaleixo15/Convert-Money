<h1>Conver Money</h1>
<p>Um conversor de moedas desenvolvido com HTML, CSS e JavaScript é um projeto clássico e essencial para quem estuda desenvolvimento web, pois combina estruturação de dados, estilização<br> de interface e lógica de programação com integração de APIs externas </p>
<hr>
<h2>Aqui está uma visão geral dos papéis de cada tecnologia nesse projeto:</h2>
<h3>1. HTML (A Estrutura)</h3>
<img width="125" height="125" alt="image" src="https://github.com/user-attachments/assets/2acbb7df-5c85-43ff-89d7-d02a729c3021" />
<p>O HTML define o "esqueleto" da aplicação. Nele, são criados os elementos essenciais:</p>
<ol>
  <li>Campos de entrada (input) para digitar o valor.</li>
  <li>Menus de seleção (select) para escolher as moedas de origem e destino (ex: BRL para USD).</li>
  <li>Botão de ação para disparar a conversão.</li>
  <li>Áreas de exibição onde o resultado final será mostrado ao usuário </li>
</ol>
<hr>
<h3>2. CSS (O Design)</h3>
<img width="130" height="130" alt="image" src="https://github.com/user-attachments/assets/01d55f50-12cc-400a-88ed-592cb9709055" />
  
<p> O CSS é responsável por tornar a ferramenta visualmente atraente e funcional. Ele cuida do:</p>
<ol>
  <li>Layout: Posicionamento dos elementos (geralmente usando Flexbox ou Grid) para que o conversor seja centralizado e responsivo.</li>
  <li>Estética: Cores, fontes, bordas arredondadas e sombras para dar um aspecto moderno e profissional.</li>
  <li>Experiência do Usuário (UX): Efeitos de hover nos botões e estados de foco nos campos de texto .</li>
</ol>
<hr>
<h3>3. JavaScript (A Inteligência)</h3>
<img width="140" height="140" alt="image" src="https://github.com/user-attachments/assets/8f103b2c-0f6b-4ada-bc0f-c5ebb39a920a" />

<p> O JavaScript é o motor do projeto. Suas principais funções incluem:</p>
<ol>
  <li>Manipulação do DOM: Capturar os valores que o usuário digita e atualizar o texto do resultado na tela.</li>
  <li>Consumo de API: Para que o conversor seja útil em 2026, ele não deve ter valores fixos ("hardcoded").<br> O JavaScript faz uma requisição (geralmente via fetch) para APIs de câmbio (como AwesomeAPI ou ExchangeRate-API) para obter as cotações em tempo real</li>
  <li>Cálculo Lógico: Realizar a operação matemática de multiplicar o valor de entrada pela taxa de câmbio recebida.</li>
</ol>
<h2>Por que este projeto é importante?</h2>
<hr>
<p>Além de ser uma ferramenta útil para o dia a dia, construir esse sistema ensina o desenvolvedor a lidar com assincronismo (esperar a resposta de um servidor) <br> e com a dinâmica da web, onde os dados mudam constantemente conforme o mercado financeiro</p>
